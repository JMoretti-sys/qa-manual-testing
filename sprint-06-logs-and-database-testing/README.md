# Sprint 06 – Logs e Banco de Dados (QA Backend)

Este projeto apresenta atividades de QA Backend realizadas em ambiente Linux e PostgreSQL, com foco em análise de logs, identificação de erros e validação de dados para uma aplicação de táxi.

## Atividades Realizadas
- Análise de logs Apache via terminal Linux
- Identificação de requisições por intervalo de IP
- Separação de erros HTTP 400 e 500
- Validação de dados em banco PostgreSQL
- Análise de volume de viagens por empresa
- Classificação de condições climáticas

## Técnicas Utilizadas
- Análise de logs
- Testes negativos
- Validação de regras de negócio
- Testes em banco de dados

## Ferramentas
- Linux (SSH)
- Bash (grep, wc)
- PostgreSQL
- SQL (JOIN, GROUP BY, HAVING, CASE)

## Evidências
Os comandos executados, consultas SQL e resultados detalhados estão documentados nos arquivos da pasta deste projeto.

## Resultado
As análises permitiram identificar padrões de erro, validar dados críticos do sistema e apoiar a investigação de possíveis falhas em backend.
