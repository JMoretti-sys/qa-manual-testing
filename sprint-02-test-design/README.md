# Sprint 02 – Design de Testes

## Objetivo
Desenvolver a documentação de design de testes utilizando técnicas clássicas de QA para garantir uma cobertura eficiente dos cenários de validação.

## Projeto
**Urban Routes – Formulário de Carteira de Motorista**

## Escopo dos Testes
Nesta sprint, o foco foi a criação de casos de teste a partir da análise de requisitos do formulário, aplicando técnicas de design de testes para identificar cenários válidos e inválidos.

## Técnicas Utilizadas
- Classes de Equivalência  
- Análise de Valores Limite (Boundary Value Analysis)

## Atividades Realizadas
- Análise dos campos do formulário de carteira de motorista  
- Identificação de entradas válidas e inválidas  
- Definição de classes de equivalência  
- Criação de casos de teste baseados em valores limite  

## Artefatos Gerados
- Casos de teste documentados  
- Tabelas de classes de equivalência  
- Tabelas de valores limite  

## Ferramentas Utilizadas
- Google Sheets / Microsoft Excel
